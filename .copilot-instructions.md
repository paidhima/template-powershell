# PowerShell Template: Copilot Language Profile

This profile extends the global baseline and applies to PowerShell repositories.

Tooling defaults
- Use `PSScriptAnalyzer` for static checks.
- Use `Pester` for tests.
- For script syntax validation, use PowerShell parser checks on changed scripts.

Required checks before completion
1. Run ScriptAnalyzer on changed PowerShell paths.
2. Run parser validation on changed scripts.
3. Run relevant Pester tests.

Typical commands
- `Invoke-ScriptAnalyzer -Path src/ -Recurse -WarningAction Stop`
- `[System.Management.Automation.Language.Parser]::ParseFile('<path>', [ref]$null, [ref]$null) | Out-Null`
- `Invoke-Pester -Path tests/ -Verbose`

Coding rules
- Use approved verb-noun names for public functions.
- Keep public and private function boundaries explicit.
- Follow repository naming and style conventions.
- Avoid broad refactors outside task scope.

Documentation rules
- Keep comment-based help for public functions when required by the repo.
- Update README/docs when behavior, prerequisites, or operational workflows change.
- Do not mass-update help blocks in untouched code.

Branch and PR policy
- Use `feature/*`, `fix/*`, or `refactor/*` branches for substantive changes.
- Treat the default branch (main/master) as protected unless user explicitly directs otherwise.

Security and operations
- Never hardcode credentials, keys, or tokens.
- Do not log sensitive values.
- Prefer reversible operational changes and explicit validation steps.
