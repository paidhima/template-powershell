# PowerShell Project Template: Copilot Instructions

## Project Context
This is a PowerShell project using:
- **Linting**: PSScriptAnalyzer (Microsoft.PowerShell.ScriptAnalyzer module)
- **Testing**: Pester v5 framework
- **Module structure**: Public functions exported; private functions prefixed with underscore
- **Documentation**: Inline help (comment-based help blocks) + README.md

## Development Workflow

### Required checks before submitting code
1. **PSScriptAnalyzer**: `Invoke-ScriptAnalyzer -Path src/ -Recurse -WarningAction Stop` (must pass with no errors)
2. **Pester tests**: `Invoke-Pester -Path tests/ -Verbose` (all tests must pass; new functions require new tests)
3. **Help content**: All public functions must have comment-based help blocks

### Quick commands
- `Invoke-ScriptAnalyzer -Path src/ -Recurse` — Check script health
- `Invoke-Pester -Path tests/ -Verbose -FailOnSkipped` — Run all tests; fail if any skipped
- `Get-Help My-PublicFunction -Detailed` — Show help for a function

## Function Naming and Conventions
- **Public functions**: PascalCase verb-noun: `Get-Data`, `Set-Configuration`, `Invoke-Sync`
- **Private functions**: `_PrivateFunctionName` (leading underscore; not exported)
- **Parameters**: PascalCase: `$ComputerName`, `$FilePath`, `$Force`
- **Variables**: camelCase for local scope: `$localData`, `$isValid` (lowercase)

## Comment-Based Help Format (Required for all public functions)
```powershell
<#
.SYNOPSIS
Short description of what the function does.

.DESCRIPTION
Longer description explaining behavior, use cases, and important side effects.

.PARAMETER ComputerName
The name of the computer to query. Accepts pipeline input.

.PARAMETER Force
If specified, bypasses confirmation prompts.

.EXAMPLE
Get-Data -ComputerName "SERVER01" -Force

This example retrieves data from SERVER01 without confirmation.

.OUTPUTS
[System.Collections.Hashtable]

Returns a hashtable with keys 'Status' and 'Data'.

.NOTES
Requires elevation. Logs all operations to event log.

.LINK
about_MyModule
#>
function Get-Data {
    [CmdletBinding()]
    param(
        [Parameter(Mandatory, ValueFromPipeline)]
        [string[]]$ComputerName,
        
        [switch]$Force
    )
    # Implementation...
}
```

## Pester v5 Test Structure
```powershell
# tests/Public/Get-Data.Tests.ps1

BeforeAll {
    # Import the module
    $ModulePath = Join-Path $PSScriptRoot "../../src/MyModule/MyModule.psm1"
    Import-Module $ModulePath -Force
}

Describe "Get-Data" {
    Context "When called with valid parameters" {
        It "Returns a hashtable with Status and Data keys" {
            $result = Get-Data -ComputerName "LOCALHOST"
            $result | Should -BeOfType [hashtable]
            $result.Keys | Should -Contain "Status"
            $result.Keys | Should -Contain "Data"
        }
    }
    
    Context "When ComputerName does not exist" {
        It "Throws an error" {
            { Get-Data -ComputerName "INVALID_HOST" } | Should -Throw
        }
    }
}
```

## Documentation
- **README.md**: Module purpose, installation, quick start example, required prerequisites
- **Help blocks**: All public functions must have full comment-based help
- **CHANGELOG.md**: Track versions and breaking changes
- **docs/ folder** (if needed): Architecture, advanced examples, troubleshooting

## Security and Privacy
- **Credentials**: Never hardcode credentials; accept as parameters or use PSCredential objects.
- **Error logs**: Avoid logging sensitive data (passwords, tokens, PII).
- **Execution policy**: Document minimum execution policy required (RemoteSigned, AllSigned, etc.).
- **Module signing**: If distributing, consider code signing.

## Version Constraints
- **PowerShell**: Target PowerShell 5.1+. Call out version-specific features in help docs.
- **Dependencies**: Pin module versions in psd1 manifest RequiredModules.

## Out-of-Scope Rules
- **No unrelated refactors** unless explicitly requested.
- **No restructuring** of test files without justification.
- **No breaking changes** to public API without major version bump and CHANGELOG entry.
- **No external dependencies** without discussion and documentation.

## Completion Criteria
A task is done when:
- [ ] All changes pass `Invoke-ScriptAnalyzer -Path src/ -Recurse` with zero errors
- [ ] All tests pass: `Invoke-Pester -Path tests/ -Verbose`
- [ ] New functions have full comment-based help blocks
- [ ] New public functions are exported in .psd1 FunctionsToExport
- [ ] New tests created for new functions or logic
- [ ] Code follows PascalCase/camelCase conventions
- [ ] Commit message is clear and references issue (if applicable)
- [ ] Changes scoped to the stated task (no unrelated refactors)
